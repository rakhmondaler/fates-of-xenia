## 🔧 Доработки (делаю я) — до отправки

### 1. Три плоскости на входе + онбординг
Сейчас скилл по умолчанию западный. Меняем: при первом обращении скилл предлагает выбрать плоскость и держится её, пока Ксения не переключит.
- **Западная** (Плацидус) — её привычная
- **Ведическая / «физическая»** (джйотиш)
- **Интегральная** — синтез двух

Онбординг при первом `/fates` объясняет эти три плоскости простыми словами.
🔸 *Решить: лейбл ведической плоскости для Ксении — «ведическая» или твоё «физическая»?*

### 2. Полировка голоса и формата
- Убрать длинное тире из ответов (твоё humanizer-правило).
- Настроить формат ответов под её вкус.
🔸 *Решить: длина по умолчанию (короткий / средний / развёрнутый), эмодзи (да/нет), заголовки (да/нет).*

### 3. Честно пересобрать распределение по стихиям
Посчитать по фактическим позициям карты, вернуть аккуратную разбивку в `portrait-western.md` и в Notion.

### 4. Дневник событий
Скилл сохраняет значимые вопросы и события Ксении (с датами) в локальный журнал, чтобы со временем видеть паттерны и точнее делать сенсмейкинг.
⚠️ *Журнал должен переживать обновления (установщик чистит папку скилла) — хранить рядом с `.venv` и не затирать. Журнал только локальный, в репозиторий не уходит.*

### 5. Йоги (раджа / дхана)
Добавить разбор сильных сочетаний карты — йоги на статус, деньги, мудрость.
🔸 *Решить: ты пришлёшь список йог из своего софта (авторитетно, как делали с картой) ИЛИ я выявлю классические по правилам, а ты сверишь?*

## 📤 Твои шаги — перед отправкой Ксении
1. Ответить на три вопроса выше (лейбл ведической, формат ответов, источник йог).
2. Если выбрал свой вариант — прислать список йог из софта.
3. Сделать **read-only fine-grained токен** на репозиторий (Contents: Read).
4. Вписать токен в `install.sh` → `REPO_URL`.
5. `cp install.sh install.command && chmod +x install.command`
6. Отправить Ксении `install.command` + два шага из `instructions-ksenia.md`.
